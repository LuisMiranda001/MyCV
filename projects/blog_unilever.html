Project Summary


📌 Project context

This project was developed as an internal solution for Unilever, with the goal of standardizing and automating the file upload and validation process used by different business teams.
The main challenge was ensuring that uploaded files complied with specific format and content rules while providing clear and immediate feedback to the user.


🛠️ Implemented solution

A web-based portal was built to allow users to upload files in a simple and intuitive way.
The frontend, developed using HTML, CSS, and JavaScript, leverages AJAX to communicate asynchronously with a Python (Flask) backend.
During the processing stage, users see a CSS-based loading indicator, avoiding page reloads and significantly improving usability.
The backend performs real-time validations on the uploaded files, checking structure, content, and business-specific rules.


🔧 Key Learnings

This project helped to:

  • - Improve user experience through asynchronous processing
  • - Reduce errors when handling critical business data
  • - Seamlessly integrate frontend, backend, and cloud services
  • - Automate notifications and minimize manual operational tasks
