Project Summary


📌 Project context

At Unilever, data teams operate multiple Databricks Workspaces across different environments (DEV, QA, and PROD).
Over time, notebooks and jobs become unused, increasing costs, operational risk, and data governance challenges.

When a cleanup initiative was planned to remove unused assets and restrict access, a common enterprise concern emerged:
👉 once notebooks or jobs are removed, users eventually claim that something critical was deleted.
The initial idea was to back up notebooks manually, which was not scalable considering there were 9 workspaces per environment.


🛠️ Implemented solution

I designed and implemented a Python-based automation using Databricks REST APIs, authenticating through workspace API keys.

The script automatically retrieves:
- All notebooks
- Notebooks metadata
- Notebook creator information

This solution allowed backing up an entire environment in approximately 30 minutes, covering DEV, QA, and PROD without manual intervention.


💰 Cost Monitoring & Optimization

As part of the initiative, I also leveraged Databricks to develop a cost monitoring dashboard to:
- Track total costs by environment
- Detect cost spikes
- Analyze the root cause behind increases
This supported proactive optimization and cost‑aware decision‑making.

🔧 Key Learnings

This project helped to:

  • - Build trust with technical teams and leadership
  • - Work with Databricks APIs at an operational level
  • - Implement real backup and data governance strategies
  • - Design automation for production-grade data platforms
  • - Reduce risks related to access control and cost management
